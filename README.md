## ⚠️ Что нужно настроить перед публикацией

### 1. Telegram-бот для приёма заявок

Заявки с форм сайта отправляются в Telegram-чат через API.
Чтобы это работало:

1. Откройте https://t.me/BotFather в Telegram
2. Отправьте команду `/newbot`
3. Задайте имя и username бота (например, `potolok_i_tochka_leads_bot`)
4. BotFather пришлёт строку вида `1234567890:AAHxxxxxxxxxxxxxxxxxxxxxx` — это **TG_BOT_TOKEN**
5. Напишите своему боту любое сообщение (например, `/start`)
6. Откройте в браузере: `https://api.telegram.org/bot<ВАШ_ТОКЕН>/getUpdates`
7. Найдите в ответе `"chat":{"id": ЦИФРЫ ...}` — это **TG_CHAT_ID**
8. Откройте файл [assets/main.js](assets/main.js), найдите блок `window.TG_CONFIG` (в самом начале файла)
9. Подставьте свои значения:

```js
window.TG_CONFIG = {
  TG_BOT_TOKEN: '1234567890:AAHxxxxxxxxxxxxxxxxxxxxxx',
  TG_CHAT_ID:   '123456789',
  SITE_NAME:    'Потолок и точка'
};
```

**Важно про безопасность:** токен будет виден в исходниках страницы. Это нормально для частной компании, но не публикуйте бота с правами админа в чужие чаты. Бот должен быть «частным» — только для приёма заявок.

Если хотите спрятать токен — потребуется backend. Это не покрыто текущим проектом.

### 2. Юридические данные

Откройте поиск (Ctrl+F) и замените во всех файлах:

- `[ваше ФИО]` → ИП ваше реальное ФИО
- `[ваш ИНН]` → 12 цифр
- `[ваш ОГРНИП]` → 15 цифр

Файлы, где это встречается:
- `index.html` (footer)
- `blog/*.html` (footer)
- `privacy.html` (Раздел «1. Кто мы» и в footer)
- `offer.html` (Раздел «13. Реквизиты» и footer)
- `404.html` (footer)

### 3. Реальный домен

Замените `https://potolok-i-tochka.ru` на ваш реальный домен во всех файлах. Сейчас он используется в:
- `<link rel="canonical">`
- `<meta property="og:url">`
- `sitemap.xml`
- `robots.txt`
- JSON-LD разметке Schema.org

### 4. Аналитика (опционально)

Перед закрывающим `</head>` каждой страницы можно вставить счётчик Яндекс.Метрики или Google Analytics. После подключения цель `lead` будет автоматически срабатывать при отправке заявки (логика уже встроена в [assets/main.js](assets/main.js)).

## Как локально запустить

Простой статический сайт. Просто откройте `index.html` в браузере. Но из-за абсолютных путей (`/assets/style.css`) лучше запустить локальный HTTP-сервер:

```bash
# В корне проекта
python3 -m http.server 8000

# Откройте http://localhost:8000
```

## Хостинг

Сайт — чистая статика, подойдёт:
- **GitHub Pages** (бесплатно, прямо из этого репо)
- **Netlify** (бесплатный план, авто-деплой из git)
- **Vercel** (бесплатный план)

Для GitHub Pages:
1. Settings → Pages
2. Source: Deploy from a branch
3. Branch: `master`, folder: `/ (root)`
4. Save

Через 1-2 минуты сайт будет доступен по адресу `https://<username>.github.io/<repo-name>/`.
